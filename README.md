## Устройство GIT

#### Хеш — идентификатор коммита
**Хеширование** (от англ. _hash_, «рубить», «крошить», «мешанина») — это способ преобразовать набор данных и получить их «отпечаток» (англ. _fingerprint_).

Информация о коммите — это набор данных: когда был сделан коммит, содержимое файлов в репозитории на момент коммита и ссылка на предыдущий, или **родительский** (англ. _parent_), коммит. Git хеширует (преобразует) эту информацию с помощью алгоритма **SHA-1** (от англ. _**S**ecure **H**ash **A**lgorithm_ — «безопасный алгоритм хеширования») и получает для каждого коммита свой уникальный хеш — результат хеширования.
Результат хеширования в Git — символьная строка.

Хеш обладает следующими важными **свойствами**:
- если хеш получить дважды для одного и того же набора входных данных, то результат будет гарантированно одинаковый;
- если хоть что-то в исходных данных поменяется, то хеш тоже изменится. 

Git хранит таблицу соответствий `хеш → информация о коммите`. Все хеши и таблицу Git сохраняет в служебные файлы. Они находятся в скрытой папке `.git` в репозитории проекта.
#### log
При вызове команды `git log` отображается список коммитов с описанием:
- коммит
- автор (с почтой)
- дата
- сообщение к коммиту

Исходный код самого Git тоже хранится в Git-репозитории (self-hosting).
Для просмотра сокращенного лога (в одну строку) используется команда `git log --oneline`. Сокращенный хэш также можно сипользовать в гите для обращения к коммитам. (Для сокращенного хэша расчитывается git как минимально необходимое количество для уникальности хэшей).
#### HEAD
Файл `HEAD` (англ. «голова», «головной») — один из служебных файлов папки `.git`. Он указывает на коммит, который сделан последним (то есть на самый новый). [[Подробнее про HEAD (git)]].
Если нужно передать последний коммит, то вместо его хеша можно просто написать слово `HEAD` — Git поймёт, что вы имели в виду последний коммит.
